Thứ tự đọc khuyến nghị

  • Ngày 1: đọc Overview + 方式設計書 + 機能変更対応マトリクス
  • Ngày 2: đọc DB仕様 + CRUD + 状態遷移 + アカウント状態整理
  • Sau đó chọn nhánh IK / IR / Batch theo assignment

Khi gặp câu hỏi thực tế

  • Muốn hiểu business scope: mở overview và matrix
  • Muốn hiểu dữ liệu/state: mở DB仕様, CRUD, 状態遷移
  • Muốn debug logic cụ thể: mở function detail + pattern matrix

Lưu ý cho member mới

  • File input này là bản đồ tài liệu, không phải tài liệu nghiệp vụ cuối
  • Đừng bỏ qua tài liệu common giữa IK và IR
  • Phải phân biệt setting trước/sau production switch
